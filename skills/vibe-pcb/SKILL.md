# Text → PCB (beginner + Claude Code → fab order)

A reproducible pipeline for the boards in this repo: a **module-carrier** PCB (a
small board that wires a ready-made module — XIAO ESP32-S3, a sensor breakout —
into a system). You write the intent in prose; Claude Code emits KiCad **by
script**, drives ERC/DRC to clean, reviews it, fits it in 3D, and packages the fab
order. You never hand-place a track in a GUI.

**Worked reference (copy this):** a module-carrier board under `examples/` (e.g.
[`examples/pager-buddy/pcb/`](../../examples/pager-buddy/) — *being filled in*): a
small MCU-to-sensor/display carrier, ERC-clean + **0 DRC / 0 unconnected**, with the
fab files generated. Read its `kicad/README.md` + brief alongside this skill; this
skill is the *method*, that dir
is a *filled-in example*.

## Why "by script", not the KiCad GUI

The `.kicad_sch` / `.kicad_pcb` are **outputs**. The sources of truth are two
Python generators:

| Generator | Emits | API |
|---|---|---|
| `gen_sch.py` | `*.kicad_sch` + `*.kicad_sym` + `sym/fp-lib-table` + `*.kicad_pro` | plain text S-expr writer |
| `gen_pcb.py` | `*.kicad_pcb` (placement + nets + outline + routing + 3D models) | bundled **`pcbnew`** |

Editing generators (not the GUI files) makes every change a readable diff, makes
the board regenerate identically on any machine, and lets a layout/module swap be
a small code edit instead of a re-draw. **Always edit the generators, then re-run.**

`gen_pcb.py` uses a tiny vocabulary you reuse for every board (see it for the
exact bodies): `place(lib,name,ref,val,x,y,rot,netmap,flip)` drops a footprint and
assigns its pads to nets · `mhole(ref,x,y)` a mount hole · `trk(net,layer,pts,w)` a
copper track · `via(net,p)` an F↔B via · `silk(text,x,y,size,rot)` silkscreen ·
`add_model(fp,path,offset,rot)` loads a STEP into the 3D view. Coordinates are
absolute mm taken from a pad dump.

## The loop

```
spec (prose)  ──►  generate  ──►  validate  ──►  review  ──►  3D fit  ──►  fab
 _brief.md         gen_sch.py      erc / drc     kicad-happy   STEP +      gerbers
 (net map,         gen_pcb.py      → 0/0/0       + dispositions renders     + BOM/CPL
  power, parts)                    (iterate)                                or LCEDA 3-piece
```

1. **Spec** — write/update `<proj>_brief.md`. This is *the* deliverable a beginner
   controls; everything downstream is mechanical. Use `references/spec-template.md`.
   The heart of it is the **net map** (pin-by-pin: net · module pin · GPIO · target
   pin · notes) — it is the single source of truth and is checked against later.
2. **Generate** — edit `gen_sch.py` + `gen_pcb.py` to match the spec, then run the
   check script (below). For a module/sensor swap, usually only the net map, the
   footprint `place(...)` calls, and the routing `trk/via` change.
3. **Validate** — `kicad-cli sch erc` + `pcb drc`, iterate to **0 error-severity /
   0 unconnected**. A DRC "violation" is error- *or* warning-severity: gate on errors
   (a real short — that's how the same-layer trace cross was caught); silk warnings are
   cosmetic. If a module mounts flush, also run the **belly gate** (`belly_check.py` /
   `BELLY_BOX=...`) — DRC can't see front copper under the module. Rules + war stories:
   `references/design-rules.md`.
4. **Review** — optional deep pass with **kicad-happy** (`cross_analysis` catches
   schematic↔PCB mismatches; `analyze_emc` scores EMC). Record each finding's
   disposition in `REVIEW.md` (fixed / accepted-not-a-defect). See design-rules.
5. **3D fit / review** — generate `.step` models of the module + sensor (build123d),
   load them with `add_model`, and check the board fits the enclosure (the
   **`vibe-cad`** route — `references/fab-and-3d.md`). For an **interactive** look
   with **no KiCad GUI**, `scripts/pcb_view.sh` opens one browser page with a
   **[2D layers | 3D board | Split]** switch — KiCanvas on the live `.kicad_pcb`
   (per-layer toggles + schematic) and the GLB in `<model-viewer>` (orbit/zoom), in a
   single unified UI. See `references/pcb-viewing.md`.
6. **Fab** — `scripts/fab_export.sh <proj> <hand-soldered-refs…>` packages gerbers +
   drill + CPL + BOM for **JLCPCB**, **or** use the LCEDA three-piece set (brief +
   outline DXF + BOM). Upload the gerber `.zip` at jlcpcb.com — **never import the
   KiCad project into EasyEDA** (its importer drops footprints + desyncs the netlist;
   the war story + CPL/BOM gotchas are in `references/fab-and-3d.md`). Then walk the
   **physical-verify-before-ordering** checklist.

## Run it (macOS, KiCad 10)

The generators and validators need **KiCad's bundled tools**, not system python:

```bash
# from the project's kicad/ dir (the one with gen_sch.py / gen_pcb.py):
S=skills/vibe-pcb/scripts
$S/pcb_check.sh <proj>                       # gen → ERC → DRC → render; prints the gate
BELLY_BOX="x0,y0,x1,y1" $S/pcb_check.sh <proj>   # + belly keep-out gate (flush module)
$S/pcb_view.sh <proj> &                       # interactive web review (no KiCad GUI): 2D layers | 3D board | Split
$S/fab_export.sh <proj> A1 J1                 # package JLCPCB gerber zip + CPL + BOM
# (`make check` in a project is a thin wrapper — see pcb-xiao-tile/kicad/Makefile)
```

`pcb_check.sh` locates `kicad-cli` + the bundled `python3` (the one with `pcbnew`),
regenerates, runs ERC + DRC, renders a top view, and prints the gate **split into
error- vs warning-severity** (warnings don't fail). `belly_check.py` proves no front
copper/vias under a flush module. `pcb_view.sh` opens **one interactive web page**
(no KiCad GUI) with a **[2D layers | 3D board | Split]** switch: 2D is **KiCanvas**
on the live `.kicad_pcb` (per-layer visibility toggles, objects, schematic — what
the 3D mesh can't give), 3D is the `kicad-cli` GLB in `<model-viewer>` (orbit/zoom),
both in one unified dark UI served locally. See `references/pcb-viewing.md`.
`fab_export.sh` packages the order (model-only +
DNP + hand-soldered refs filtered, Y flipped, JLC footprint names). Override tool
paths with `KICAD_CLI` / `KICAD_PY` on non-mac / other KiCad versions.

## Scaffolding a new board

Copy the reference layout (don't start from a blank KiCad project):

```
hardware/<line>/pcb-<name>/
  <name>_brief.md            # the spec (start here — references/spec-template.md)
  <name>_outline.dxf + _gen.py   # board outline (for LCEDA hand-off)
  <name>_bom.csv             # BOM (LCSC/JLC part numbers)
  kicad/
    gen_sch.py  gen_pcb.py   # the two generators (edit these)
    Makefile                 # `make check` / `make review`
    <name>.pretty/           # project footprints (custom module land)
    models/  gen_*.py *.step # build123d 3D models for the fit-check
    REVIEW.md  analysis/     # review findings + kicad-happy output
    fab/                     # gerbers + BOM/CPL + zip (the order)
    view/                    # pcb_view.sh 3D GLB (gitignored — regenerated)
```

## Hard rules (full list + the war stories in `references/design-rules.md`)

- **Edit the generators, never the `.kicad_*`.** Re-run to regenerate.
- **Power must be live out of the fab** — default solder-jumpers **closed**, not
  open (an open default ships a dead board).
- **Keep copper out from under a flush-soldered module's belly** — its exposed
  back-pads will short to carrier copper. Route under-belly nets on the *back*
  layer; keep front copper + vias in the margins. (This is the #1 carrier gotcha.)
  DRC won't catch it — verify with `scripts/belly_check.py` / `BELLY_BOX=...`.
- **Fab via Gerber upload, not a EasyEDA import** — `scripts/fab_export.sh` → upload
  the `.zip` to JLCPCB. Importing the KiCad project into EasyEDA mangles footprints +
  the netlist; it's the importer, not your board.
- **Sync schematic and PCB values** — `cross_analysis` fails on mismatches.
- **Photo/caliper-verify the real module before ordering** — pull-ups already
  present (→ leave yours DNP), VIN voltage, header line order, land pattern / row
  pitch, mount-hole spacing, I²C address. Track these in the brief's EST checklist.

## When to reach for this skill vs others

- New board / module swap / layout change → **here**.
- The **enclosure** that holds the board → `vibe-cad` (build123d), and view with
  `cad-viewer`. The board's 3D models feed that fit-check.
- Reviewing/handing off DXF or STEP to a cut/CNC house → `sendcutsend` / `dxf`.

## Keeping this current (living doc)

This skill exists to **compound** hard-won PCB experience so the next board is
easier. When using it on a new board surfaces a new gotcha, a fix, or a better
practice, **fold it back in** (SKILL.md / `references/` / `scripts/`) and commit it
with the work — don't let a lesson live only in a chat. Same for `vibe-cad`.
